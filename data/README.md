# Data Directory

This directory serves as the centralized storage for datasets, extracted metadata, and evaluation outputs produced during the JAMEX (Judicial Multi-Agent Metadata Extraction) experiment.

> **⚠️ Important Notice regarding Data Availability:** 
> For compliance with the data governance policies of the [STJ Open Data Portal](https://dadosabertos.web.stj.jus.br/), the full texts of the appellate decisions (*acórdãos*) are **not distributed herein**. This repository includes an initial set of examples and the base indices necessary to demonstrate the execution logic.
> 
> To exemplify how the extractions are performed, **only two sample documents have been extracted and included in this repository for each model and strategy** (baseline prompt and agents). Furthermore, these examples have been properly **anonymized** to remove personal data, even though they originate from public records.
> 
> The full structured dataset must be dynamically fetched on demand by executing the dataset assembly notebook (`01_notebooks/01_data_preparation.ipynb`).

---

## Directory Structure

### 1. Source Datasets
- **`espelhos_acordaos_artigo2026.parquet`**: The foundational index containing the selected sample of candidate decisions (identifiable by `id_peca` and `seq_documento_acordao`) utilized in the experiment.
- **`espelhos_acordaos_artigo2026_com_texto.parquet`**: The enriched target dataset generated after running the data preparation notebook. It correlates the fetched full-text court decisions with their gold-standard structured metadata.

### 2. Extraction Outputs
- **`espelhos_base_<model>/`**: Artefacts containing the structured metadata extracted via the single-prompt **baseline strategy**. Includes execution summary logs (`resumo_extracao.csv`) corresponding to each evaluated LLM.
- **`espelhos_agentes_<model>/`**: Artefacts produced by the **JAMEX multi-agent orchestration pipeline**. Features the extracted JSON metadata, alongside agent observability trails, error logs, and evaluation metrics by the LLM-as-a-Judge scripts.

### 3. Analysis and Evaluation
- **`analises_<experiment_setup>/`**: Directories holding aggregated quantitative metrics (F1-score, precision, recall) and comparison outcomes generated by the evaluation scripts.


## Key Analysis Files

Within each `analises_<experiment>/` folder (such as `analises_comparacao_base_gpt5`), you will find the consolidated results of the structured and semantic evaluation process. The most relevant files for an in-depth audit are:

- **Consolidated Metrics Reports (`comparacao_extracoes.csv`, `relatorio_analise.md`, and `relatorio_analise_estatistica.md`)**: Files that summarize the LLM-as-a-Judge evaluations for **Precision, Recall, and F1-Score** per field and globally. They also document textual similarity metrics (ROUGE-L, BERTScore, and SBERT) as secondary diagnostics, and detail variations in completion rates across orchestrations.
- **Charts and Boxplots (`graficos/`)**: Automatically rendered images generated during testing (e.g., `comparativo_global_score.png` and other F1-Score derivatives) that visually and comparatively demonstrate the distributions of evaluation metrics achieved by each model and extraction strategy.
- **Detailed JSON Files (`jsons/`)**: Within the analysis JSONs subfolder, each evaluated document has its isolated output. These JSON files store in detail original excerpts vs. extractions, applied scores, and raw evaluated metrics, making them ideal for manually auditing where a particular agent failed, dropped out (non-completion), hallucinated, or suppressed texts (recall/precision issues).
